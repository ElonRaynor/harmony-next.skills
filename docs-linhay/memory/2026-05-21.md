## 能力沉淀：Empty Ability SDK 适配与交互 smoke

- 复盘 GitHub issue #11 后，将可重复使用的通用能力合并回 `harmony-next` skill 文档：SDK 版本适配构建、模拟器安装启动、layout 断言和交互 smoke。
- 决策：Empty Ability 模板默认仍保留 `5.0.0(12)`；目标环境需要其他 SDK 时，在复制出的 fixture 中覆盖 `compatibleSdkVersion` 和 `targetSdkVersion`，例如 HarmonyOS 6.0.2 / API 22 使用 `6.0.2(22)`，避免把本机 SDK 状态固化成模板默认值。
- 决策：API 22 构建时 `DEVECO_SDK_HOME` 必须指向 SDK 根目录 `/Applications/DevEco-Studio.app/Contents/sdk`，不能指向 `sdk/default`。
- 决策：API 22 schema 需要保留 `AppScope/resources/base/media/app_icon.png`、app `icon`、Ability `icon` 和 `startWindowIcon`。
- 验证模式：安装启动后通过 `uitest dumpLayout` 查 `Harmony Smoke Ready` / `tapCount=0` / `smoke-increment`；再用 `uitest uiInput click` 点击按钮中心点，重新 dump 并断言 `Harmony Smoke Tapped` / `tapCount=1`。
