## DevEco 模拟器抓包代理排查

- 结论：宿主机抓包工具监听代理端口、Mac 系统代理开启时，HarmonyOS 模拟器仍不会自动把宿主机 `127.0.0.1:<proxy-port>` 系统代理继承为模拟器内代理。
- 证据：HDC target `127.0.0.1:10100` 已连接；模拟器网卡为 `10.0.2.15`，默认网关为 `10.0.2.2`；浏览器打开 HTTP 测试站点后，设备侧 `netstat` 显示从 `10.0.2.15` 直连目标站点 `:80`，没有连接 `10.0.2.2:9090` 或宿主机局域网 IP 的 `:9090`。
- 证据：清空抓包工具 session 后导出的 HAR 只包含宿主机进程流量，未出现模拟器浏览器请求；临时 HAR 文件因包含宿主机代理流量已删除。
- 后续建议：调试包应优先在应用内显式设置 `connection.setAppHttpProxy({ host: "10.0.2.2", port: 9090, exclusionList: [] })`，并在 `http.HttpRequestOptions` 中设置 `usingProxy: true`；HTTPS 明文抓包还需配置抓包工具 CA 到应用信任链，若存在 certificate pinning 则需调试环境关闭或替换 pin。
