# 2026-06-05

## Issue 15: DevEco Emulator launch diagnostics

- 处理 GitHub issue #15，并顺带覆盖 issue #14 的 image root 校验诉求。
- `hvd_manager.py` 将 DevEco build SDK root 与 emulator image root 分开：`--sdk-root` / `DEVECO_SDK_HOME` 只表示 build SDK root，`--image-root` / `HARMONY_EMULATOR_IMAGE_ROOT` 表示模拟器镜像根；macOS 常见镜像根为 `$HOME/Library/Huawei/Sdk`。
- `launch-preflight` 和 `launch` 会按 HVD `config.ini` 的 `imageSubPath` 校验 `<image-root>/<imageSubPath>`，错误目录返回 `missingConfig=["imageRootSystemImage"]`，不会启动 Emulator。
- `launch` 会落盘 `emulator-launch.log`，trace socket 超时或 Emulator 静默退出时返回 `processExitCode`、`logPath`、`logTail`、`hvdRuntime` 和 `hdcSnapshot`；HDC 连接后继续检查 `bootevent.boot.completed`，并用 `stabilityWait` 持续验证 target。
- 本机真实验证发现仅保持 trace socket 或仅 detach holder 都不够；必须同时 detach Emulator 进程和 trace holder，否则调用进程退出后 HDC target 会消失。
- 最终本机 `Codex Test Phone` 验证：`--stability-seconds 90` 通过，脚本返回后外部 `hdc list targets -v`、`param get bootevent.boot.completed`、`uitest dumpLayout` 均可用；验证后已 stop Emulator 并重启 HDC 回到 `[Empty]`。
- 验证：`python3 -m unittest harmony-next/tests/test_hvd_manager.py` 通过 18 个测试。
