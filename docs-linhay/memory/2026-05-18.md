# 2026-05-18

## DevEco Studio 模拟器免 IDE 启动

- 已验证可以不启动 DevEco Studio IDE，直接调用 Emulator 可执行文件启动 HarmonyOS 本地模拟器。
- 免 IDE 启动不是只调用 `Emulator -hvd` 即可；还需要传入 HVD 父目录、镜像根目录、trace pipe 名称，并由 runner 准备本地占位通道。占位通道实现细节不写入记忆。
- 启动成功的验收信号：Emulator 独立进程启动、日志出现 `Guest OS Boot Completed!!`、`hdc list targets` 出现 `127.0.0.1:<hdc-port>`、基础 `uitest` 命令可执行。
- AI 自动化优先走设备侧命令链路：设备发现、boot check、应用生命周期、窗口/无障碍定位、UI 自动化、日志/诊断。
- UI 自动化默认使用 `hdc + uitest`；底层输入、真实截图、真实 layout、大日志、安装卸载、清数据、录制、压测、测试入口都进入用户确认档。
- 多实例通过启动参数显式分配 HDC 端口；脚本只能操作 `Connected` 目标，多个 `Connected` 且未显式指定 target 时必须停止并让上层选择。
- 当前没有确认到稳定公开的 `Emulator create/edit/wipe` CLI。本地 HVD 创建可行，但属于复制同版本可运行实例并同步必要元数据的高风险操作；文件级步骤不写入记忆。
- HVD 手工创建不宜只写声明信息，应基于同版本可运行实例同步元数据与运行期数据，并通过 `Emulator -list -details`、启动日志和 HDC 连接状态验证。
- HVD 参数边界按当前版本表现保守处理：名称只使用字母、空格、数字、下划线且 1-60 字符；RAM/ROM 最小 2GB；ROM 最大 1023GB；手机/折叠屏 3.5-9 inch，平板 8-15 inch，PC 10-20 inch；手机/折叠屏分辨率 720-3500 px，PC 1000-4000 px；DPI 240-640。
- 非公开或半公开能力只记录命令级结论、用途、验证结果和风险等级，不记录来源路径、获取方法、内部符号、文件定位信息或业务标识。
- 失败诊断分三档：默认只读无真实附件；增强档补日志/CPU/内存/窗口/显示摘要；用户确认档才采真实截图、真实 layout、大日志或日志包。
- 技术调查阶段按用户要求不修改 `harmony-next/`，只写入 `docs-linhay/`。研发记录落到 `docs-linhay/dev/2026-05-18-deveco-emulator-cli.md`。
